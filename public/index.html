Fluxo

Do evento bruto ao aviso legivel

O projeto recebe `push` e `pull_request`, valida a assinatura do GitHub com `x-hub-signature-256`, monta um embed com contexto e pode incluir um resumo natural vindo do OpenRouter.

Stack

  • Node.js + TypeScript
  • Express
  • crypto nativo do Node
  • fetch nativo
  • Discord Webhook

Principais rotas

  • `GET /` apresenta o projeto
  • `GET /health` retorna status
  • `POST /webhooks/github` recebe eventos

Setup

Como colocar no ar em poucos passos

01

Configure o ambiente

Defina `DISCORD_WEBHOOK_URL`, `GITHUB_WEBHOOK_SECRET` e, se quiser resumo natural, `OPENROUTER_API_KEY`.

02

Expose a URL publica

Use ngrok para teste local ou publique no Render/Railway antes de cadastrar o webhook do GitHub.

03

Cadastre os eventos do GitHub

Em `Settings > Webhooks`, aponte para `/webhooks/github` e selecione `Pushes` e `Pull requests`.

04

Receba no Discord

Os embeds mostram branch, autor, commits, links do evento e o resumo IA quando a chave do OpenRouter estiver ativa.

.env

Configuracao direta, sem banco e sem bot

O backend foi desenhado para ser simples de deploy. Basta preencher as variaveis abaixo e iniciar o servidor. 

PORT=3000
DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/...
GITHUB_WEBHOOK_SECRET=seu-segredo
OPENROUTER_API_KEY=sk-or-v1-...
OPENROUTER_MODEL=google/gemini-3.1-flash-lite
